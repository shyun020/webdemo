웹 서비스 구축 후 템플릿 및 SSL 적용

☝🏻 STEP ONE : 서비스 구축 [WEB]

웹(Web)은 인터넷을 통해 정보를 공유하기 위한 하나의 수단이며 World Wide Web의 약어에요. 인터넷 상에서 문서와 같은 리소스를 공유하기 위한 정보 시스템을 의미해요.

사용자는 웹 브라우저를 통해 웹 사이트에 접속하여 원하는 정보를 찾고, 웹 서버는 이를 요청에 따라 제공하는 역할을 해요. 이러한 구조를 통해, 인터넷을 통해 정보를 손쉽게 검색하고 공유할 수 있어요.

✌🏻 STEP TWO : 웹 템플릿 적용

Document Root

→ Document root는 웹 서버에서 웹 페이지 파일들이 위치한 디렉터리를 의미해요. 이 디렉터리는 웹 서버가 루트 디렉터리로부터 웹 페이지를 찾아갈 때 기준이 되는 시작점이에요.

리눅스에서는 대표적으로 Apache 웹 서버의 Document root 디렉터리는 "/var/www/html"에요. 이 디렉터리에 웹 페이지 파일들을 저장하고, Apache 서버가 이 디렉터리를 기준으로 웹 페이지를 서비스해요. 따라서, Document root 디렉터리는 웹 서버에서 매우 중요한 역할을 해요.

좀 더 쉽게 “웹 서버는 Document Root 경로의 있는 파일을 읽어온다.” 라고 생각하면 편할 것 같아 요. Docoment Root가 /var/www/html/인데, /home/soohyun 디렉터리에 웹 페이지 파일들이 위치 하면 웹 서버는 서비스 할 수 없어요.

Index.html, Index.php & 우선순위

→ Index.html과 Index.php는 둘 다 웹 페이지의 첫 페이지인 홈페이지로 자주 사용되는 파일명 이에요. 하지만, 둘은 서로 다른 파일 형식이에요.

Index.html은 HTML (Hypertext Markup Language)로 작성된 정적 웹 페이지에요. HTML은 웹 페이지의 내용을 구성하는 데 사용되는 언어에요. HTML 파일은 브라우저에서 직접 열 수 있고, CSS 및 JavaScript와 같은 다른 언어로 작성된 코드를 참조하여 웹 페이지를 동적으로 만들 수 있어요.

반면에, Index.php는 PHP (Hypertext Preprocessor)로 작성된 동적 웹 페이지에요. PHP는 서버 측 스크립트 언어로, HTML로 작성된 정적 페이지와 달리, 동적 컨텐츠를 생성하고 데이터베이스와 상호 작용하는 데 사용돼요. PHP 파일은 웹 서버에서 처리되어 브라우저에게 보내지기 전에 PHP 코드가 실행돼요.

따라서, Index.html은 정적인 페이지를 작성할 때 사용되고, Index.php는 동적인 페이지를 작성할 때 사용돼요.

더불어 “우선순위”라는 개념도 있어요. 웹 서버에서 디렉토리 내에 Index 파일이 있다면, 일반적으 로 웹 브라우저는 그 파일을 자동으로 열어서 보여줘요. Index 파일의 이름은 서버 설정에서 변경 할 수 있어요.

Index 파일의 우선순위는 설정 파일에 의해 결정돼요. 일반적으로 Index.html이 가장 먼저 우선 순위를 갖고, 그 다음으로 Index.htm, Index.php, Index.asp 등의 파일이 우선순위를 가지게 돼요. 이러한 우선순위는 설정 파일에서 변경할 수 있어요.

🤟🏻 STEP THREE : SSL 인증서 적용

SSL 인증서

→ SSL (Secure Sockets Layer) 인증서는 웹사이트에서 전송되는 정보를 암호화하고 인증하는 데 사용 되는 디지털 인증서에요. SSL 인증서는 인증기관 (CA, Certificate Authority) 에서 발급 받아 사용자 의 브라우저와 웹 서버 간에 암호화된 연결을 설정하게 돼요.

SSL 인증서를 사용하면 사용자와 웹 서버 간의 통신이 안전하게 암호화되므로 중요한 정보, 예를 들어 개인 정보, 신용 카드 정보 등이 노출되는 것을 방지할 수 있어요. 인증서는 일정 기간마다 갱신해야 하며, 갱신된 인증서는 이전 인증서를 대체하여 사용해요.

웹사이트를 운영할 때 SSL 인증서를 적용하면 웹사이트가 안전하게 운영될 수 있으며, 브라우저에 서도 해당 웹사이트가 안전하다는 것을 인증할 수 있어요. 이를 통해 사용자들의 개인정보 보호와 보안성을 향상 시킬 수 있어요.

🔒 SSL 인증서 적용

Let’s Encrypt

→ Let's Encrypt는 인터넷에서 SSL/TLS 인증서를 무료로 발급하고 관리할 수 있도록 하는 인증 기관(CA)입니다. Let's Encrypt는 HTTPS를 쉽게 사용하도록 하여 보안성을 높이는 것을 목표로 하고 있습니다.

Let's Encrypt는 무료이기 때문에 일반적으로 유료로 제공되는 인증서와는 달리, 모든 사용자가 쉽게 이용할 수 있습니다. Let's Encrypt에서 발급한 인증서는 대부분의 웹 브라우저에서 신뢰할 수 있기 때문에, HTTPS를 사용하는 웹사이트에 대한 신뢰성을 높이는 데에 큰 역할을 합니다.

또한, Let's Encrypt는 자동화된 방식으로 인증서를 발급하고 갱신할 수 있도록 도와줍니다. 이를 통해 사용자는 인증서 관리에 대한 부담을 덜 수 있고, 보안적인 측면에서도 안정성을 높일 수 있습니다.
